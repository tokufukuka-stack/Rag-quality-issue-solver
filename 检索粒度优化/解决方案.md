# RAG 检索粒度优化解决方案

## 问题描述（原文）

**检索粒度不当——chunk 太大则噪声多，太小则语境丢失**

在 RAG 系统中，文档切分（chunking）是影响检索质量的核心环节。Chunk 过大时，向量嵌入会包含大量无关信息，导致检索噪声增加、匹配精度下降；Chunk 过小时，单个片段缺乏上下文，语义表达不完整，生成阶段难以产出高质量回答。这一"粒度困境"是目前 RAG 工程实践中最普遍的挑战之一。

---

## 解决方案

### 方案一：父子分层检索（Parent-Child / Small-to-Big Retrieval）

**核心思路：**
将文档同时切分为大粒度的"父 chunk"和小粒度的"子 chunk"。嵌入和检索阶段使用小 chunk（精准匹配），但返回给 LLM 生成阶段时替换为对应的大 chunk（丰富上下文）。

**实现方法：**
1. 将文档切分为父 chunk（500–2000 token，代表段落或章节）
2. 再将每个父 chunk 切分为更小的子 chunk（100–300 token）
3. 仅对子 chunk 建立向量索引
4. 检索时按子 chunk 相似度召回，再映射回父 chunk 提供给 LLM

**参考来源：**
- [Parent-Child Chunking in LangChain for Advanced RAG (Medium)](https://medium.com/@seahorse.technologies.sl/parent-child-chunking-in-langchain-for-advanced-rag-e7c37171995a)
- [Parent-Child Retriever | GraphRAG](https://graphrag.com/reference/graphrag/parent-child-retriever/)
- [Advanced Chunking Strategies for RAG (AddAxis)](https://addaxis.ai/advanced-chunking-strategies-for-rag/)

---

### 方案二：混合粒度路由（Mix-of-Granularity, MoG）

**核心思路：**
借鉴 Mixture-of-Experts 思想，用一个路由模型（router）根据查询内容动态判断最佳检索粒度（句子级、段落级、章节级等），而非为所有查询使用固定 chunk 大小。

**实现方法：**
1. 对同一文档建立多粒度索引（细粒度到粗粒度）
2. 训练轻量路由模型，以软标签损失函数学习查询-粒度映射关系
3. 查询时由路由模型决定使用哪一粒度的索引层
4. 扩展版本 MoG-Graph（MoGG）将文档预处理为图结构，支持跨位置片段检索

**参考来源：**
- [Mix-of-Granularity: Optimize the Chunking Granularity for RAG (ArXiv 2406.00456)](https://arxiv.org/abs/2406.00456)
- [ACL Anthology COLING 2025 版本](https://aclanthology.org/2025.coling-main.384.pdf)

### 方案三：语义感知切分（Semantic Chunking）

**核心思路：**
不依赖固定 token 数量切分，而是通过计算相邻句子的语义相似度，在"语义断点"处切分，确保每个 chunk 内容语义连贯。

**实现方法：**
1. 将文本分割为单独句子
2. 为每对相邻句子计算嵌入相似度（如余弦相似度）
3. 在相似度显著下降处（即语义转折点）进行切分
4. 可配合 LlamaIndex 的 `SemanticSplitterNodeParser` 或 LangChain 的 `SemanticChunker` 实现

**参考来源：**
- [Chunking Strategies for RAG: Best Practices (Unstructured)](https://unstructured.io/blog/chunking-for-rag-best-practices)
- [Chunking Strategies to Improve LLM RAG Pipeline Performance (Weaviate)](https://weaviate.io/blog/chunking-strategies-for-rag)
- [Best Chunking Strategies for RAG in 2026 (Firecrawl)](https://www.firecrawl.dev/blog/best-chunking-strategies-rag)

### 方案四：自适应分块策略（Adaptive Chunking）

**核心思路：**
基于文档本身的内在特征（而非固定规则）自动选择最合适的切分方式。该方法通过衡量文档的参考完整性、块内凝聚度、上下文一致性等指标来决策。

**实现方法：**
1. 对文档计算五个内在指标：
   - References Completeness（RC）：参考引用的完整性
   - Intrachunk Cohesion（ICC）：块内语义凝聚度
   - Document Contextual Coherence（DCC）：文档上下文连贯性
   - Block Integrity（BI）：结构块完整性
   - Size Compliance（SC）：尺寸合规性
2. 根据指标组合选择最适合该文档的切分策略
3. 临床决策支持场景下，该方法实现了 87% 的准确率（对比基线 50%）

**参考来源：**
- [Adaptive Chunking: Optimizing Chunking-Method Selection for RAG (ArXiv 2603.25333)](https://arxiv.org/abs/2603.25333)
- [Comparative Evaluation of Advanced Chunking for RAG in Clinical Decision Support (PMC)](https://pmc.ncbi.nlm.nih.gov/articles/PMC12649634/)

---

### 方案五：元切分（Meta-Chunking）

**核心思路：**
通过模型自身的困惑度（Perplexity）或边际采样（Margin Sampling）来识别文本中的最优分割点，同时在切分时保留全局信息，实现语义完整的动态切分。

**实现方法：**
1. **困惑度切分（Perplexity Chunking）**：在困惑度突变处切分，代表语义转折
2. **边际采样切分（Margin Sampling Chunking）**：基于模型预测概率的边际值识别切分点
3. 双策略结合保证局部精准分割与全局信息保留

**参考来源：**
- [Meta-Chunking: Learning Text Segmentation and Semantic Completion via Logical Perception (ArXiv 2410.12788)](https://arxiv.org/abs/2410.12788v3)
- [HiChunk: Evaluating and Enhancing RAG with Hierarchical Chunking (ArXiv)](https://arxiv.org/html/2509.11552)
