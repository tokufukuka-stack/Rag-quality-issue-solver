# Top-K 截断导致关键信息被排除在候选之外

## 问题描述

在 RAG（检索增强生成）系统中，Top-K 截断是指检索阶段只保留相似度得分最高的 K 个文档片段（chunk），并将其传入 LLM 生成答案。当 K 值设置过小时，真正相关的文档可能因排名靠后而被截断排除；当 K 值过大时，又会引入大量噪声，占用 LLM 上下文窗口，导致"注意力稀释"和"Lost in the Middle"效应。静态固定 K 值无法适应不同查询的复杂度差异，是 RAG 系统召回质量的核心瓶颈之一。

---

## 解决方案

### 方案一：自适应 K 值选择（Adaptive-k）

**核心思路**

不使用固定的 Top-K，而是动态检测文档相似度得分序列中的最大"跌落间隙"（Gap）。将相似度得分降序排列后，找到相邻得分之间下降最大的位置，以此为截断点，只保留该点之前（高分区）的文档。这一方法基于"相关文档之间得分紧密聚集，不相关文档得分骤降"的经验观察。

**实现方法**

1. 初始检索较多候选（如 Top-50 或 Top-100）；
2. 对所有候选的相似度得分降序排列；
3. 计算相邻得分之间的差值 `delta[i] = score[i] - score[i+1]`；
4. 找到 `delta` 最大的位置 `i*`；
5. 仅保留前 `i*+1` 个文档传入 LLM。

**参考来源**

- [Adaptive-k Retrieval: A Smarter Way to Trim Context in RAG（Medium）](https://indranildchandra.medium.com/adaptive-k-retrieval-a-smarter-way-to-trim-context-in-rag-687a399a67fe)
- [Beyond Top-k: How Adaptive-k Dynamically Selects the Best Context for RAG（DataSci Ocean）](https://datasciocean.com/en/paper-intro/adaptive-k/)

### 方案二：两阶段检索 + 重排序（Two-Stage Retrieval with Reranking）

**核心思路**

第一阶段使用向量近似搜索宽泛召回（如 Top-50 ～ Top-100），保证高召回率；第二阶段使用 Cross-Encoder 重排序模型（如 Cohere Rerank、BGE Reranker、RankRAG）对候选进行精细打分，再取 Top-K 传入 LLM。Cross-Encoder 能同时考虑查询和文档的完整语义交互，评分质量显著优于 Bi-Encoder 向量相似度。

**实现方法**

1. 向量数据库初步检索 Top-50 候选；
2. 使用 Cross-Encoder 对每个 `(query, chunk)` 对打分；
3. 按新分数降序排列，取 Top-5 ～ Top-10 传入 LLM；
4. 可选：设置最低置信度阈值，低于阈值的片段直接丢弃。

**推荐工具**

- Cohere Rerank API
- `cross-encoder/ms-marco-MiniLM-L-6-v2`（HuggingFace）
- NVIDIA NeMo Reranking
- BGE-Reranker（BAAI）

**参考来源**

- [Rerankers and Two-Stage Retrieval（Pinecone）](https://www.pinecone.io/learn/series/rag/rerankers/)
- [Enhancing RAG Pipelines with Re-Ranking（NVIDIA Technical Blog）](https://developer.nvidia.com/blog/enhancing-rag-pipelines-with-re-ranking/)
- [RankRAG: Unifying Context Ranking with Retrieval-Augmented Generation（NeurIPS 2024）](https://arxiv.org/html/2407.02485v1)

### 方案三：基于聚类的自适应检索（Cluster-based Adaptive Retrieval, CAR）

**核心思路**

将检索到的候选文档在嵌入空间中进行聚类分析，检测"高相关文档密集簇"向"低相关文档稀疏区"的过渡点，以此作为自适应截断边界。不同于简单的分数跌落，CAR 考虑了文档之间的空间分布结构，能更鲁棒地处理得分平缓下降的场景。

**实现方法**

1. 初始检索 Top-N 候选（N 较大）；
2. 对候选文档的嵌入向量进行层次聚类或 K-Means 聚类；
3. 识别第一个"高密度相关簇"，将该簇的边界作为截断点；
4. 仅将该簇内文档传入 LLM。

**预期效果**（论文数据）

- LLM token 消耗减少 60%
- 端到端延迟降低 22%
- 幻觉率降低 10%

**参考来源**

- [Cluster-based Adaptive Retrieval: Dynamic Context Selection for RAG（Arxiv 2511.14769）](https://arxiv.org/abs/2511.14769)

### 方案四：动态 RAG（DynamicRAG）

**核心思路**

将 K 值的选择建模为一个可学习的决策过程。训练一个专门的"文档数量决策器"（Document Number Decider），该决策器根据查询的复杂度、当前检索结果的信息量等特征，动态输出应传入 LLM 的文档数量。这种方法将原本人工调参的静态 K 值转变为端到端可优化的动态参数。

**实现方法**

1. 训练一个轻量级分类器或使用 LLM-as-judge 评估查询复杂度；
2. 根据复杂度预测最优 K 值区间（如简单问题 K=3，复杂问题 K=10）；
3. 检索对应数量的文档传入生成器。

**参考来源**

- [DynamicRAG: Leveraging Outputs of Large Language Models（Arxiv 2505.07233）](https://arxiv.org/pdf/2505.07233)
- [Optimizing Retrieval Augmentation with Dynamic Top-K Tuning（Medium）](https://medium.com/@sauravjoshi23/optimizing-retrieval-augmentation-with-dynamic-top-k-tuning-for-efficient-question-answering-11961503d4ae)

### 方案五：查询扩展（Query Expansion）+ 假设文档嵌入（HyDE）

**核心思路**

Top-K 截断问题有时根源在于查询表达不够准确，导致相关文档在向量空间中距离偏远、排名靠后。通过查询扩展（生成多个语义变体查询并合并检索结果）或 HyDE（让 LLM 先生成一个假设性答案文档，用该假设文档的嵌入去检索真实文档），可以从源头提升召回率，减少因表达差异导致的关键信息遗漏。

**实现方法（HyDE）**

1. 将原始查询输入 LLM，生成一段假设性答案文本；
2. 对假设答案文本进行嵌入，得到"假设文档向量"；
3. 用假设文档向量替代（或融合）原始查询向量进行检索；
4. 将检索结果传入 LLM 生成最终答案。

**参考来源**

- [How Query Expansion (HyDE) Boosts Your RAG Accuracy（Chitika）](https://www.chitika.com/hyde-query-expansion-rag/)
- [Advanced Query Transformations to Improve RAG（Towards Data Science）](https://towardsdatascience.com/advanced-query-transformations-to-improve-rag-11adca9b19d1/)
- [Optimizing Query Generation for Enhanced Document Retrieval in RAG（Arxiv）](https://arxiv.org/html/2407.12325v1)
