# 语义漂移解决方案

## 问题描述

**原文：** 语义漂移——查询意图与检索到的文档语义不一致

语义漂移（Semantic Drift）是 RAG 系统中最常见的检索质量问题之一。用户的查询意图与检索到的文档在语义上不匹配，导致检索结果偏离用户真实需求。核心原因包括：查询与文档之间存在"词汇鸿沟"（vocabulary gap）、嵌入空间中查询向量与文档向量的分布不对称、以及复杂查询的意图难以被单一向量准确表达。

---

## 主要解决方案

### 方案一：HyDE（Hypothetical Document Embeddings）假设文档嵌入

**核心思路：**
利用 LLM 根据用户查询生成一个"假设性答案文档"，然后将该假设文档转化为向量嵌入进行检索，而非直接使用原始查询进行检索。这样做的好处是将"问题-文档"匹配转化为"文档-文档"匹配，显著缩小语义空间差距。

**实现方法：**
1. 用户提交查询 → LLM 生成 k 个假设性文档（无需真实准确，只需语义相近）
2. 对每个假设文档计算嵌入向量
3. 取平均或分别检索，得到候选文档集合
4. 可结合 reranker 进一步精排

**适用场景：** 查询较短或抽象（如"如何解决性能问题"），与文档的表述方式差异大。

**参考来源：**

- [Haystack Documentation - HyDE](https://docs.haystack.deepset.ai/docs/hypothetical-document-embeddings-hyde)

### 方案二：Hybrid Search（混合检索）结合 Reranking（重排序）

**核心思路：**
将稠密向量检索（semantic search）与稀疏检索（BM25 等关键词匹配）结合，取两者的优势互补。稠密检索捕捉语义相似性，稀疏检索确保关键词精确匹配。然后通过 Cross-Encoder Reranker 对候选集进行精排，进一步提升相关性。

**实现方法：**
1. 并行执行 BM25 检索 + 向量语义检索
2. 使用 Reciprocal Rank Fusion (RRF) 或加权融合策略合并结果
3. 用 Cross-Encoder 模型（如 ms-marco-MiniLM）对 top-N 候选进行重排序
4. 返回最终 top-K 结果

**效果数据：** 混合检索 + 重排序可将 top-10 相关文档命中率从 62%（纯 BM25）/ 71%（纯语义检索）提升至 87%。

**参考来源：**
- [Superlinked - Optimizing RAG with Hybrid Search & Reranking](https://superlinked.com/vectorhub/articles/optimizing-rag-with-hybrid-search-reranking)
- [Pinecone - Rerankers and Two-Stage Retrieval](https://www.pinecone.io/learn/series/rag/rerankers/)
- [AIMultiple - Hybrid RAG: Boosting RAG Accuracy in 2026](https://research.aimultiple.com/hybrid-rag/)

### 方案三：Query Rewriting & Decomposition（查询改写与分解）

**核心思路：**
使用 LLM 在检索前对用户查询进行改写、扩展或分解，使查询更精确地表达用户意图，或将复杂查询拆分为多个子查询分别检索。

**实现方法：**
1. **Query Rewriting：** LLM 将用户原始查询改写为更适合检索的形式（添加同义词、消除歧义）
2. **Query Expansion：** 扩展查询词，增加相关术语
3. **Query Decomposition：** 将复杂问题拆分为多个独立子问题，分别检索后合并结果
4. **Step-back Prompting：** 先让 LLM 生成更高层次的抽象问题，再检索

**适用场景：** 复杂查询、多意图查询、口语化或模糊表述的查询。

**参考来源：**
- [Advanced RAG 06: Exploring Query Rewriting](https://aiexpjourney.substack.com/p/advanced-rag-06-exploring-query-rewriting-23997297f2d1)

### 方案四：Adaptive RAG（自适应 RAG）

**核心思路：**
训练一个轻量级分类器，根据查询的复杂度自动选择不同的检索策略。简单查询走快速通道（直接检索），复杂查询走增强通道（多步检索、查询分解、多源融合）。

**实现方法：**
1. 训练 query complexity classifier（使用 T5-large 等模型）
2. 将查询分为三个层级：简单（直接检索）、中等（单步检索+重排）、复杂（多步检索+推理）
3. 根据分类结果动态路由到对应的检索 pipeline
4. 各 pipeline 可组合 HyDE、混合检索、重排等技术

**参考来源：**
- [Meilisearch - Adaptive RAG Explained](https://www.meilisearch.com/blog/adaptive-rag)
- [Let's Data Science - Agentic RAG Self-Correcting Retrieval](https://letsdatascience.com/blog/agentic-rag-self-correcting-retrieval)

### 方案五：RAG Fusion（RAG 融合检索）

**核心思路：**
对用户查询自动生成多个变体查询（不同措辞、不同角度），分别检索后使用 Reciprocal Rank Fusion 合并排名，以覆盖更多语义角度，减少单一查询表述导致的语义漂移。

**实现方法：**
1. 用户查询 → LLM 生成 3-5 个查询变体
2. 每个变体独立执行向量检索
3. 使用 RRF 算法融合多个检索结果的排名
4. 可选：增加 reranker 精排步骤

**效果数据：** 在工业部署中，RAG Fusion 相比单查询检索在多样性和召回率上均有显著提升。

**参考来源：**
- [Arxiv - Scaling Retrieval Augmented Generation with RAG Fusion](https://arxiv.org/html/2603.02153)
- [DEV Community - RAG Is Not Dead: Advanced Retrieval Patterns](https://dev.to/young_gao/rag-is-not-dead-advanced-retrieval-patterns-that-actually-work-in-2026-2gbo)
