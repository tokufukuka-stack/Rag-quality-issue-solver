# RAG 幻觉问题解决方案

## 问题描述（原文）

> 幻觉——模型无视检索结果，凭训练知识编造内容

在 RAG（检索增强生成）系统中，LLM 有时会忽视检索到的相关上下文，转而依赖自身训练时的参数化知识生成回答，导致输出内容与检索文档不符，甚至与事实相悖。这种现象称为"忠实度幻觉"（Faithfulness Hallucination）或"上下文-记忆冲突"（Context-Memory Conflict）。研究表明，即使检索质量很高，模型仍可能进行"后理由化"（post-rationalization）——即先生成一个看似合理的答案，再把检索来源作为装饰性引用附上，实际并未基于检索内容推理。

---

## 主要解决方案

### 方案一：Self-RAG（自反思检索增强生成）

**方案名称：** Self-RAG — 自适应检索与自我批评框架

**核心思路：**
训练语言模型在生成过程中自主判断"是否需要检索"、"检索结果是否被利用"以及"生成内容是否受检索支撑"，通过插入特殊反思标记（reflection tokens）将检索决策内嵌于生成流程。

**实现方法：**
1. 在训练数据中注入 `[Retrieve]`、`[Relevant]`、`[Supported]`、`[Utility]` 等特殊控制标记；
2. 微调 LLM，使其在生成前判断是否触发检索，在生成中评估检索片段是否被使用，在生成后评分答案的文档支撑度；
3. 推理时根据自我评分选择最优生成路径。

**参考来源：**
- [Self-RAG: Learning to Retrieve, Generate, and Critique through Self-Reflection (Asai et al., 2023) — arXiv:2310.11511](https://arxiv.org/abs/2310.11511)
- [Hallucination Mitigation Review — MDPI Mathematics 2025](https://www.mdpi.com/2227-7390/13/5/856)

### 方案二：Corrective RAG（CRAG）—— 纠正式检索重写

**方案名称：** CRAG — 检索质量评估 + 补偿性重检索

**核心思路：**
在生成之前，对检索到的文档质量打分；若分数低于阈值，自动触发"补偿性检索"（如 Web 搜索），并在文档知识提炼（Knowledge Refinement）后再交给 LLM 生成，从源头保证生成依据的可靠性。

**实现方法：**
1. 部署一个轻量评估器（evaluator）对每个检索结果打分：Correct / Ambiguous / Incorrect；
2. 当检索结果评为 Ambiguous 或 Incorrect 时，激活 Web 搜索或二次检索；
3. 对获取的文档做知识提炼（去噪、分割），再构造 Prompt 发送给生成模型；
4. 可与 LangGraph 等编排框架集成，实现动态路由。

**参考来源：**
- [CRAG: Corrective Retrieval Augmented Generation (Yan et al., 2024) — arXiv:2401.15884](https://arxiv.org/abs/2401.15884)
- [RAG Hallucinations Explained: Causes, Risks, and Fixes — Mindee Blog](https://www.mindee.com/blog/rag-hallucinations-explained)

### 方案三：Chain-of-Verification（CoVe）—— 验证链推理

**方案名称：** CoVe — 链式验证问答，逐步核实生成内容

**核心思路：**
模型先生成初步回答，再自动生成一系列"验证问题"（verification questions），对初步回答中的每个关键声明单独核实，最后综合核实结果输出修订后的答案。独立问答形式能显著降低自我确认偏差。

**实现方法：**
1. **草稿生成**：LLM 根据问题和检索上下文生成初步答案；
2. **规划验证**：LLM 自动分解草稿中的关键事实，生成独立的验证子问题；
3. **独立执行**：对每个子问题单独调用模型（或检索+模型），避免受草稿影响；
4. **最终修订**：将所有验证结果汇总，生成修订后的最终回答。

**参考来源：**
- [Chain-of-Verification Reduces Hallucination in LLMs (Dhuliawala et al., 2023) — arXiv:2309.11495](https://arxiv.org/abs/2309.11495)
- [Improving AI-Generated Responses: Techniques for Reducing Hallucinations — The Learning Agency](https://the-learning-agency.com/the-cutting-ed/article/hallucination-techniques/)

### 方案四：EvidenceRL —— 基于强化学习的证据一致性训练

**方案名称：** EvidenceRL — 以 NLI 为奖励信号的 RL 对齐训练

**核心思路：**
使用自然语言推理（NLI）模型作为奖励函数，通过强化学习（RL）微调 LLM，使模型在生成时天然地优先遵循检索证据而非参数记忆，从训练层面根治"忽视上下文"问题。

**实现方法：**
1. 使用 Focus-Then-Verify 流程构建（前提,假设）对：以检索文档片段为前提，以生成声明为假设；
2. 用冻结的 NLI 交叉编码器对每对打分（支持/中立/矛盾）；
3. 以 NLI 得分为奖励信号，通过 PPO 等 RL 算法更新生成模型参数；
4. 与 RLHF 管线兼容，可叠加在 SFT 基础上继续训练。

**参考来源：**
- [EvidenceRL: Reinforcing Evidence Consistency for Trustworthy Language Models — arXiv:2603.19532](https://arxiv.org/html/2603.19532)
- [Correctness is not Faithfulness in RAG — arXiv:2412.18004](https://arxiv.org/pdf/2412.18004)

### 方案五：Prompt 层约束 + 忠实度评估后处理

**方案名称：** 提示工程约束 + 自动忠实度评分过滤

**核心思路：**
通过精心设计的系统提示，明确要求 LLM 只能基于所提供的检索内容作答，不得引用外部知识；同时在输出后使用独立评估模型（如 RAGAS、DeepEval）对答案进行忠实度评分，低分结果触发重新生成或人工审核。

**实现方法：**
1. **Prompt 约束**：在 System Prompt 中加入明确指令，如"你只能基于以下文档回答，若文档中没有相关信息，请明确说明'文档中未提及'"；
2. **引用强制**：要求模型为每个陈述附上来源文档编号（如 [doc1]），并在后处理中验证引用是否真实存在；
3. **忠实度评分**：使用 RAGAS 框架计算 Faithfulness 指标，或使用 GPT-4 作为评判裁判（LLM-as-Judge）；
4. **重生成触发**：设置忠实度阈值，低于阈值的回答自动触发带更严格约束的二次生成。

**参考来源：**
- [Measuring LLM Groundedness in RAG Systems — deepset Blog](https://www.deepset.ai/blog/rag-llm-evaluation-groundedness)
- [RAG LLM Prompting Techniques to Reduce Hallucinations — Galileo AI Blog](https://galileo.ai/blog/mastering-rag-llm-prompting-techniques-for-reducing-hallucinations)
- [RAGAS Framework — GitHub](https://github.com/explodinggradients/ragas)
