# 对抗样本攻击解决方案

## 问题描述（原文）

**对抗样本攻击——精心构造的文档可以操纵检索排名**

攻击者通过向 RAG 知识库中注入精心构造的文档（对抗性段落），使这些文档在特定查询下获得异常高的检索排名，从而诱导模型生成攻击者预设的错误回答，或干扰正常检索结果。此类攻击包括：知识库投毒（Knowledge Poisoning）、检索排名操控（Retrieval Ranking Manipulation）、拒绝服务攻击（DoS via Blocker Documents）等变体。

---

## 主要解决方案

### 方案一：多层防御框架 RAGuard

**方案名称：** RAGuard（检索感知的分层防御）

**核心思路：**
将防御分为两层：第一层在检索侧对 Retriever 进行对抗性训练，使其对投毒文档不敏感；第二层在推理侧通过零知识推断补丁（Zero-Knowledge Inference Patch）识别对推理结果有异常因果影响的文档并过滤。两层防御协同作用，互相补充。

**实现方法：**
1. 对 Dense Retriever（如 DPR）进行对抗性微调，在训练数据中加入对抗样本，增强模型对异常文档的鲁棒性。
2. 在推理阶段，对每个候选文档单独计算其对最终答案的"因果影响力"——若某文档的加入/去除导致答案大幅偏移，则将其标记为可疑文档并排除。
3. 结合两层机制，在 NQ 和 BEIR 等基准测试中显著降低攻击成功率，同时保持干净数据的检索精度。

**参考来源：**
- [RAGuard: A Layered Defense Framework for Retrieval-Augmented Generation](https://openreview.net/pdf/f7bd605a521e7fe5a23b34a2ef51659d26264d62.pdf)

### 方案二：轻量级对抗段落检测 RAGDefender

**方案名称：** RAGDefender（基于聚类的实时对抗段落检测）

**核心思路：**
不依赖额外 LLM 推理或模型重训练，通过对检索结果进行快速语义聚类来识别异常的对抗性段落。对抗文档往往在语义空间中孤立于正常文档，因此聚类离群点即为可疑对象。

**实现方法：**
1. **单跳问题**：对 Top-K 检索结果进行 K-Means 或层次聚类，将与主流类别语义距离过大的段落视为对抗样本剔除。
2. **多跳问题**：采用"浓度分析"（Concentration-based Grouping）——正常段落在语义上具有多样性，而对抗段落往往集中在某一特定子空间，通过统计密度异常来检测。
3. 整个过程为轻量级 ML 推断，无需调用 LLM，延迟极低。

**参考来源：**
- [Efficient Defense Against Knowledge Corruption Attacks on RAG (RAGDefender)](https://kevinkoo001.github.io/assets/pdf/acsac25-ragdefender.pdf)
- [Rescuing the Unpoisoned: Efficient Defense against Knowledge Corruption Attacks](https://arxiv.org/pdf/2511.01268)

### 方案三：图重排序防御 GRADA

**方案名称：** GRADA（Graph-based Reranking Against Adversarial Documents）

**核心思路：**
将检索到的文档集合建模为图结构，节点为文档段落，边权重为文档间的语义相似度。通过图算法（如 PageRank 或图神经网络）计算每个节点的"可信度中心性"，排名靠前但与其他正常文档高度不一致的对抗文档会因低中心性被降权。

**实现方法：**
1. 构建文档相似度图：对 Top-K 文档两两计算余弦相似度，构建加权无向图。
2. 运行 Personalized PageRank 或谱聚类，识别与图主连通分量不一致的孤立节点。
3. 对孤立节点（对抗文档）进行降分重排序，确保语义一致的文档集群获得更高权重后送入 LLM。

**参考来源：**
- [GRADA: Graph-based Reranking against Adversarial Documents in RAG](https://aclanthology.org/2025.emnlp-main.1132.pdf)

### 方案四：文档溯源验证 RAGShield

**方案名称：** RAGShield（基于溯源验证的深度防御）

**核心思路：**
为知识库中的每个文档赋予密码学可验证的"溯源证明"（Provenance），只有能通过溯源验证的文档才会进入检索候选，从根源上阻断未经授权的对抗文档注入。

**实现方法：**
1. 知识库摄取（Ingestion）阶段，对每个文档计算哈希签名并记录来源元数据（作者、时间戳、来源 URL）。
2. 检索阶段，在将文档传入 LLM 前验证其签名链；签名不匹配或来源不明的文档被拦截。
3. 对五类攻击威胁级别（包括持有签名密钥的自适应攻击者）实现 0.0% 攻击成功率。
4. 特别适合政府、企业等对知识库有严格访问控制要求的场景。

**参考来源：**
- [RAGShield: Provenance-Verified Defense-in-Depth Against Knowledge Base Poisoning](https://arxiv.org/html/2604.00387)

### 方案五：一致性多数投票 ReliabilityRAG

**方案名称：** ReliabilityRAG（基于图论的一致性多数检测）

**核心思路：**
从信息论和图论角度出发，寻找检索文档集合中的"一致性多数"（Consistent Majority）。当知识库受到部分投毒时，正常文档在信息上会形成多数一致，而对抗文档是少数异常者。通过图论方法识别并隔离该少数群体。

**实现方法：**
1. 将检索文档两两比较，计算答案一致性分数，构建一致性图。
2. 使用最大团（Maximum Clique）或 k-core 分解找到一致性最高的文档子集作为可信集合。
3. 仅使用可信集合内的文档生成最终回答，提供理论上的鲁棒性保证（Provably Robust）。

**参考来源：**
- [ReliabilityRAG: Effective and Provably Robust Defense for RAG-based Web-Search](https://arxiv.org/pdf/2509.23519)
- [ReliabilityRAG (OpenReview)](https://openreview.net/pdf?id=D9JeNTs5Bu)
