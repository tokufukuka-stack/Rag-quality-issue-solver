# PDF/扫描件解析错误 — 表格、公式、图片信息大量丢失

## 问题描述（原文）

> PDF/扫描件解析错误，表格、公式、图片中的信息大量丢失

在 RAG 系统中，文档预处理阶段通常需要将 PDF 或扫描件转换为可检索的文本。然而，传统解析器（如 PyPDF2、pdfminer）在面对以下情况时会发生严重信息丢失：
- **扫描 PDF**：图像层文字无法直接提取，需 OCR；
- **表格**：行列关系被展平为乱序文本；
- **数学公式**：LaTeX/MathML 符号被忽略或乱码；
- **嵌入图片/图表**：完全丢失，无法进入索引。

---

## 解决方案

### 方案一：使用专用 AI 驱动的文档解析库——Docling

**核心思路**
IBM 开源的 Docling 采用两个专门训练的 AI 模型：DocLayNet（布局识别）和 TableFormer（表格结构识别），将 PDF 的视觉布局信息转化为结构化输出。

**实现方法**
```python
from docling.document_converter import DocumentConverter

converter = DocumentConverter()
result = converter.convert("document.pdf")
markdown_output = result.document.export_to_markdown()
```
- 输出格式支持 Markdown 和 JSON；
- 自带 OCR（基于 EasyOCR/Tesseract），处理扫描件；
- 表格以 Markdown 表格形式保留行列结构；
- 可直接集成 LangChain、LlamaIndex、Haystack；
- 本地运行，不依赖外部 API，无 GPU 要求。

**参考来源**
- [Docling 官网](https://www.docling.ai/)
- [PDF Data Extraction Benchmark 2025: Docling vs Unstructured vs LlamaParse](https://procycons.com/en/blogs/pdf-data-extraction-benchmark/)
- [PDF Table Extraction Showdown: Docling vs. LlamaParse vs. Unstructured](https://boringbot.substack.com/p/pdf-table-extraction-showdown-docling)

### 方案二：Unstructured.io 管道化解析

**核心思路**
Unstructured 提供分阶段的文档解析管道，包含版式分析、表格检测、OCR 和元素分类，在跨文档类型的综合 Benchmark 中整体得分领先（表格分 0.844）。

**实现方法**
```python
from unstructured.partition.pdf import partition_pdf

elements = partition_pdf(
    filename="document.pdf",
    strategy="hi_res",           # 高精度模式，启用版式模型
    infer_table_structure=True,  # 推断表格结构
    extract_images_in_pdf=True,  # 提取嵌入图片
)
for el in elements:
    print(el.category, el.text)
```
- `hi_res` 策略使用 detectron2 版式模型；
- 输出元素类型包括 Table、Image、Formula、NarrativeText 等；
- 企业版支持云端 API，开源版可本地部署。

**参考来源**
- [Unstructured Leads in Document Parsing Quality: Benchmarks Tell the Full Story](https://unstructured.io/blog/unstructured-leads-in-document-parsing-quality-benchmarks-tell-the-full-story)
- [Best PDF Extractor: LlamaParse vs Unstructured vs Vectorize](https://www.chitika.com/best-pdf-extractor-rag-comparison/)

### 方案三：LlamaParse — 云端智能解析服务

**核心思路**
LlamaIndex 提供的 LlamaParse 使用 LLM 辅助的文档理解，尤其擅长半结构化文档（如财务报告、医疗记录）中的表格语义提取，可将表格转为 Markdown 同时保留上下文关系。

**实现方法**
```python
from llama_parse import LlamaParse
from llama_index.core import SimpleDirectoryReader

parser = LlamaParse(
    result_type="markdown",
    premium_mode=True,     # 启用高精度模式
    num_workers=4,
)
documents = SimpleDirectoryReader(
    "./docs", file_extractor={".pdf": parser}
).load_data()
```
- 输出 Markdown，保留表格、标题层级、代码块；
- 支持多语言 OCR；
- 需要 LlamaCloud API Key（有免费额度）；
- 在金融/医疗领域表格上精度优于 Docling。

**参考来源**
- [RAG + LlamaParse: Advanced PDF Parsing for Retrieval](https://medium.com/kx-systems/rag-llamaparse-advanced-pdf-parsing-for-retrieval-c393ab29891b)

### 方案四：多模态 VLM 页面级解析（Hybrid Multimodal Parsing）

**核心思路**
将 PDF 每页渲染为高分辨率图像，再用视觉语言模型（VLM，如 GPT-4o、Gemini 2.5 Pro）直接"阅读"页面图像，输出结构化 Markdown，天然处理公式、图表、表格、手写体。

**实现方法**
```python
import base64, openai
from pdf2image import convert_from_path

pages = convert_from_path("document.pdf", dpi=200)
client = openai.OpenAI()

results = []
for page_img in pages:
    # 将页面图像转为 base64
    buf = io.BytesIO()
    page_img.save(buf, format="PNG")
    b64 = base64.b64encode(buf.getvalue()).decode()

    response = client.chat.completions.create(
        model="gpt-4o",
        messages=[{
            "role": "user",
            "content": [
                {"type": "image_url", "image_url": {"url": f"data:image/png;base64,{b64}"}},
                {"type": "text", "text": "Extract all text, tables (as markdown), and formulas (as LaTeX) from this page."}
            ]
        }]
    )
    results.append(response.choices[0].message.content)
```
- 完全绕过传统 PDF 解析器；
- 对公式的 LaTeX 重建能力突出；
- 成本较高（每页约 $0.01–0.05）；
- 适合高价值文档（学术论文、技术手册）。

**参考来源**
- [The Best Way to Parse Complex PDFs for RAG: Hybrid Multimodal Parsing — Instill AI](https://www.instill-ai.com/blog/the-best-way-to-parse-complex-pdfs-for-rag-hybrid-multimodal-parsing)
- [Vision-Guided Chunking Is All You Need: Enhancing RAG with Multimodal Document Understanding (arxiv)](https://arxiv.org/html/2506.16035v2)

### 方案五：RAG-Anything — 多模态 All-in-One RAG 框架

**核心思路**
香港大学开源的 RAG-Anything 框架将文本、图片、表格、公式统一纳入 RAG 管道，为每种内容类型设计专用索引与检索策略，并通过统一的查询接口返回跨模态结果。

**实现方法**
参见官方 GitHub：安装后可对混合内容 PDF 建立多模态索引，支持按内容类型分通道检索再融合排序。

**参考来源**
