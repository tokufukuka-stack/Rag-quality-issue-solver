# 层级结构丢失解决方案

## 问题描述（原文）

**层级结构丢失——章节、标题、小节的层次关系被抹平**

在文档处理过程中，原始文档往往具有清晰的层次结构（如一级标题 > 二级标题 > 三级标题 > 正文段落）。传统的 RAG 流程在对文档进行 chunk 切分时，常常将这些层级关系完全丢弃，导致：

- chunk 脱离其所属的上下文语境（例如某段文字属于哪个章节）
- 检索到的片段无法让模型理解其在文档中的位置
- 同一术语在不同章节下含义不同，但模型无从区分
- 跨小节的逻辑关联无法被正确捕获

---

## 主要解决方案

### 方案一：分层父子 Chunk 策略（Hierarchical Parent-Child Chunking）

**方案名称：** 父子节点分层索引

**核心思路：**
构建两级甚至多级 chunk 体系。父 chunk 保留完整的章节/小节内容，子 chunk 是从中进一步切分的细粒度片段。检索时先匹配子 chunk，返回时扩展为其父 chunk 以提供完整语境。

**实现方法：**
- **LangChain**：使用 `ParentDocumentRetriever`，将大块父文档存入 `docstore`，将小块子文档向量化后存入 `vectorstore`。检索命中子 chunk 后自动回溯父 chunk 返回给 LLM。
- **LlamaIndex**：使用 `HierarchicalNodeParser`，将文档解析为多级节点（如 2048 → 512 → 128 token），并通过 `AutoMergingRetriever` 在检索时判断是否将多个相邻子节点合并为父节点返回。

```python
# LlamaIndex 示例
from llama_index.node_parser import HierarchicalNodeParser, get_leaf_nodes
node_parser = HierarchicalNodeParser.from_defaults(chunk_sizes=[2048, 512, 128])
nodes = node_parser.get_nodes_from_documents(documents)
leaf_nodes = get_leaf_nodes(nodes)
```

### 方案二：结构感知文档解析 + 层级元数据注入（Structure-Aware Parsing with Metadata Breadcrumbs）

**方案名称：** 面包屑路径元数据注入

**核心思路：**
在对文档进行 chunk 切分时，为每个 chunk 附加其所属的完整层级路径（如"第一章 > 1.2 节 > 1.2.3 小节"），以结构化元数据的形式存入索引。检索时可基于此元数据进行过滤或排序。

**实现方法：**
- 使用 Markdown 解析器（如 `MarkdownHeaderTextSplitter`）识别标题层级
- 将标题路径拼接为 `breadcrumb` 字段存入 chunk 元数据
- 在向量检索后，利用 metadata filtering 精确定位章节范围
- 可结合 Unstructured.io 或 Azure AI Document Intelligence 实现自动结构提取

```python
# LangChain MarkdownHeaderTextSplitter 示例
from langchain.text_splitter import MarkdownHeaderTextSplitter
headers_to_split_on = [("#", "H1"), ("##", "H2"), ("###", "H3")]
splitter = MarkdownHeaderTextSplitter(headers_to_split_on=headers_to_split_on)
splits = splitter.split_text(markdown_text)
# 每个 split 的 metadata 中自动包含 H1/H2/H3 路径
```

### 方案三：HiQA 层级上下文增强（Hierarchical Contextual Augmentation RAG）

**方案名称：** HiQA 级联元数据增强

**核心思路：**
HiQA 是一种专为多文档问答设计的层级 RAG 架构，包含三个核心模块：
1. **Markdown 格式化器**：将原始文档统一转换为 Markdown 格式，保留标题层级
2. **层级上下文增强器**：提取每个 chunk 所在的完整标题链（H1→H2→H3），将其作为 cascade metadata 附加到 chunk
3. **多路由检索器**：根据查询类型选择不同路由（精确匹配、语义检索、元数据过滤）

**实现方法：**
- 文档转 Markdown（可用 Pandoc、Docling、Marker 等工具）
- 递归解析 Markdown AST 提取标题层级
- 为每个段落 chunk 生成形如 `"文档名 > 第一章 > 第2节 > 内容"` 的 augmented text 用于 embedding
- 检索时可使用 cascade context 提升召回准确率

**参考来源：**
- [HiQA: A Hierarchical Contextual Augmentation RAG for Multi-Documents QA (arXiv:2402.01767)](https://arxiv.org/html/2402.01767v2)
- [Document Hierarchy in RAG: Boosting AI Retrieval Efficiency](https://medium.com/@nay1228/document-hierarchy-in-rag-boosting-ai-retrieval-efficiency-aa23f21b5fb9)

### 方案四：递归文档结构解析（Recursive Document Parsing）

**方案名称：** 递归章节感知切分

**核心思路：**
按照文档的自然结构进行递归切分：先按一级标题分割成大块，再在每一块内按二级标题切分，依此类推，直到 chunk 大小满足 token 限制。这样每个最终 chunk 天然具有层级归属。

**实现方法：**
- **LangChain `RecursiveCharacterTextSplitter`**：配置分隔符列表 `["\n# ", "\n## ", "\n### ", "\n\n", "\n", " "]`，优先按高级标题切分
- **LlamaIndex `SimpleNodeParser` + 自定义 splitter**：在节点解析时保留 `section` 信息
- 可结合 `docling`（IBM 开源文档理解框架）或 `unstructured` 库自动识别文档结构元素

**参考来源：**
- [Chunking Strategies to Improve LLM RAG Pipeline Performance | Weaviate](https://weaviate.io/blog/chunking-strategies-for-rag)
- [Mastering Document Chunking Strategies for RAG | Medium](https://medium.com/@sahin.samia/mastering-document-chunking-strategies-for-retrieval-augmented-generation-rag-c9c16785efc7)

### 方案五：视觉文档理解（Vision-Guided Document Parsing）

**方案名称：** 多模态结构感知解析

**核心思路：**
对于 PDF、Word 等富格式文档，使用视觉语言模型（VLM）或专用的文档理解模型（如 LayoutLM、Docparser、Azure Document Intelligence）先理解文档的视觉布局，再提取层级结构信息，从而比纯文本解析更准确地识别标题层级。

**实现方法：**
- 使用 **Azure AI Document Intelligence** 的 Layout 模型，识别段落、标题、表格等结构元素，并返回每个元素的层级标注
- 使用 **RAG-Anything**（HKUDS 开源项目）处理多模态文档，保留图文混排时的结构信息
- 使用 **Docling**（IBM Research 开源）将 PDF 转换为带结构标注的 Markdown/JSON

**参考来源：**
- [Vision-Guided Chunking Is All You Need: Enhancing RAG with Multimodal Document Understanding (arXiv)](https://arxiv.org/html/2506.16035v2)
- [Build Intelligent RAG with Image understanding & hierarchical document structure analysis with Azure](https://medium.com/@manoranjan.rajguru/build-intelligent-rag-with-image-understanding-hierarchical-document-structure-analysis-with-aefeb3ed47ea)
